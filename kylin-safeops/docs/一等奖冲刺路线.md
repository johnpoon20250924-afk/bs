# KylinSafeOps 一等奖冲刺路线

## 当前定位

项目已经具备前后端演示、Tool Contract、安全自检、审计回放、openKylin no-install 验证脚本，以及 CPU/Memory/Disk 真实采集能力。后续目标不是堆页面，而是补齐评委会直接追问的三条主线：

1. MCP 标准兼容层是否真实存在。
2. Agent 是否能跨场景做根因分析。
3. 安全审计是否能完整证明每次动作可控、可追溯、可复盘。

## 已推进能力

- 新增 MCP 工具注册表：`backend/app/execution/tool_registry.py`
- 新增 MCP 兼容 API：`backend/app/api/routes_mcp.py`
- 新增 FastAPI 路由：`/api/mcp`
- 支持接口：
  - `GET /api/mcp/tools/list`
  - `POST /api/mcp/tools/call`
  - `POST /api/mcp` JSON-RPC，支持 `initialize`、`tools/list`、`tools/call`
- MCP 调用会经过：
  - Tool Registry
  - MCP Gateway Policy
  - Tool Contract
  - Environment Adapter
  - Tool Execution
  - Audit Persist
- 审计记录新增 `audit_chain`，导出报告新增 `Audit Chain` 段落。
- RCA 候选根因已扩展：
  - `port_conflict`
  - `cpu_pressure`
  - `memory_pressure`
  - `disk_pressure`
  - `config_error`
  - `permission_denied`
- Agent 诊断入口已接入多场景 PlanSpec 路由：
  - `resource_pressure_diagnosis`
  - `network_port_diagnosis`
  - `log_anomaly_diagnosis`
  - `service_failure_diagnosis`

## 一等奖核心差距

### P0

1. MCP 协议深度还需要继续加强  
当前已完成兼容入口和工具列表/调用，但还需要补充更接近真实 MCP 客户端的测试样例、协议说明和答辩截图。

2. RCA 场景需要继续泛化  
当前已完成 service、network、resource、log 四类 PlanSpec 基础模板。下一步需要增加每类场景的专用证据图谱、阈值策略和答辩样例。

3. openKylin 实测证据必须补齐  
需要在 openKylin Desktop V2.0 SP2 上运行 no-install 验证脚本，保存命令输出、截图、录屏和报告。

4. Prompt Injection 防护需要接入主链路  
红队自检已有样例，但自然语言诊断入口还需要显式记录输入净化和注入检测结果。

### P1

- 增加 MCP 调用的前端测试入口。
- 增加 `/api/agent/diagnose` 的场景识别。
- 增加 CPU/Memory/Disk 阈值配置。
- 增加审计报告一键导出入口说明。
- 增加非 root 运行证明和 systemd user service 示例。

### P2

- 增加可视化推理链路回放。
- 增加 MCP 工具市场式展示。
- 增加银河麒麟与 openKylin 的差异适配说明。
- 增加模型 critic 的可插拔配置。

## 下一阶段建议顺序

1. 完成 MCP 协议测试脚本和文档。
2. 把 Prompt Injection 检测接入 `/api/agent/diagnose`。
3. 为 service、network、resource、log 四类模板补专用证据图谱。
4. 在 openKylin 上跑 `scripts/kylin_immutable_verify.sh` 并回传报告。
5. 打包答辩演示链路：dashboard、MCP 调用、审计导出、openKylin 真实采集截图。
