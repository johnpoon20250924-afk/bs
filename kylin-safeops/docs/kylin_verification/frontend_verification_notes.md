## 审计中心 UI 重叠问题修复记录

此前在 openKylin/Firefox 较小窗口下，审计中心页面存在明显布局问题：

- 下方“执行时间线”“证据摘要”“决策与结论”区域与上方会话列表/详情区域重叠。
- 页面高度没有被内容正确撑开。
- “决策与结论”区域在窄内容区中被裁切或横向溢出。

已修复：

- 调整审计中心主容器行布局，避免依赖不稳定的百分比高度计算。
- 将会话列表、详情区域、表格区域改为明确的 grid 分区与内部滚动。
- 调整底部审计区域在 1500px 以下的响应式布局：时间线和证据摘要保持两列，“决策与结论”独占下一整行。
- 将“决策与结论”流程图改为可换行布局，避免横向裁切。
- 增加移动/窄屏断点，避免按钮、标签页和详情卡片溢出。

涉及文件：

```text
frontend/src/styles.css
```

## 重新截图验证要求

修复后需要重新打开审计中心页面，并分别在以下窗口尺寸截图：

| 窗口尺寸 | 验证重点 |
| --- | --- |
| `1366 x 768` | 下方三块区域不再与上方会话列表/详情区域重叠；“决策与结论”不裁切。 |
| `1440 x 900` | 审计中心整体可滚动，主区和底部区上下分区清晰。 |
| `1920 x 1080` | 宽屏下底部区域可充分展开，页面无横向溢出。 |

推荐截图：

- 审计中心顶部指标和会话列表/详情区域。
- 向下滚动后截图“执行时间线”“证据摘要”“决策与结论”完整区域。
- 若使用 Firefox/openKylin，需要确认右侧“决策与结论”文字完整可见。

## 当前结论

前端可在 openKylin 2.0 SP2 中打开并展示核心页面。审计中心 UI 重叠问题已在前端样式层修复，后续应使用 openKylin/Firefox 按上述三种窗口尺寸重新截图，作为最终比赛材料中的审计中心展示证据。
